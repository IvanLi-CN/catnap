# UI 线框图（Wireframes）

本目录提供本计划的 UI 信息架构与页面线框图，供主人快速对齐交互与信息展示口径（不绑定具体 UI 框架实现）。

页面：

- `inventory-monitor.svg`：库存监控（网格布局：一行一个可用区，每行内为配置网格；默认展开，可折叠）
- `products.svg`：全部产品（分组视图 + 监控开关；云服务器不支持监控）
- `settings.svg`：系统设置（通知参数、站点地址、查询频率与抖动）
- `logs.svg`：日志（过滤 + 分页）
- `unauthorized-401.svg`：401 未授权页

通用交互约定：

- 顶部导航固定四模块：库存监控 / 全部产品 / 系统设置 / 日志。
- 导航视觉口径：默认（非选中/非悬浮）背景透明、无描边；选中项用左侧强调条 + 更粗字体（避免“按钮感”与界面噪音）。
- 统一在应用启动时调用 `GET /api/bootstrap` 获取：国家地区、可用区域、配置列表 + 当前用户设置与监控状态。
- “库存监控”页面顶栏右侧（在“最近刷新”右边）提供“重新同步”按钮：强制抓取上游并刷新（30s 限流），同步中显示进度（x/y），失败时提示可关闭后恢复默认状态。
- 同源：UI 与 API 同源托管；跨域不支持。
- 鉴权失败：页面级展示 `401`（不泄露鉴权细节）。

设计原则（为“安静、可读、可维护”服务）：

- 少说“内部实现”：UI 文案只表达用户可理解/可操作的信息；不展示 `fid`、内部 flag、接口路径等实现细节。
- 避免“无意义内容”：不放假数据占位卡；当列表为空/列数不足时，用空状态或自适应列数承接。
- 视觉降噪：默认态不做按钮感（导航/标签不加描边与底色）；用留白与层级区分分区，描边仅用于必要的交互控件。
- 一致性优先：同一层级标题/正文/辅助文字字号与字重一致；同一行的主要动作/次要动作尽量等高等宽并对齐。

版式与间距（统一网格与节奏）：

- 8pt grid：所有坐标与尺寸尽量落在 8 的倍数上。
- 外边距：`24`
- 主区结构：Top bar（64）+ 左侧导航（240）+ 主内容（其余），横向间距 `24`
- 容器内边距：`24`
- 组件间距：`16`（同组），`24`（跨组/分区）
- 列表/表格行高：`40`（含分隔线）

页面特定约定：

- `inventory-monitor.svg`：一行一个可用区；行内配置采用自适应列数：`count=1 => 1 列`，`count=2 => 2 列`，`count>=3 => 3 列`。

临时预览（PNG）：

- `../tmp/ui-previews/*.png`：由 SVG 渲染生成，便于快速查看；主人确认后可删除该目录。
